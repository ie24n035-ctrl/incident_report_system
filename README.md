# クライアント 05 — インシデント報告

中規模工場・安全管理担当

### 状況
インシデントを紙の用紙に記録してバインダーに保管している。

### 問題
誰も分析しない。同じ場所で同じ事故が繰り返されている。

### 依頼
「インシデントをデジタルで報告して、同じ問題が繰り返されていたら教えてくれるシステムがほしい。」

### 追加依頼
 「重大事故はメールで通知してほしい。」

### ディレクトリ構成と役割

```text
├── static/
│   └── style.css          # 画面のデザイン・レイアウトを整えるCSSファイル
├── templates/
│   ├── add.html           # インシデントを新規登録・報告する画面. ※インシデント=重大な事件や事故（アクシデント）には至らなかったものの、業務の正常な進行を妨げたり、将来的に大きな問題に発展したりする可能性のある「予期せぬ出来事」や「トラブル」
│   ├── edit.html          # 過去のインシデント内容を編集・修正する画面
│   └── index.html         # インシデント一覧および重複アラートを表示するメイン画面. 登録された情報の並べ替え(日付順・重要度順)や、複数発生しているトラブルの自動警告アラート、編集・削除へのリンクが配置
├── README.md              # 本書（プロジェクトの説明書）
├── app.py                 # データの処理・画面遷移・重複分析を行うアプリ本体（Python）
├── incidents.db           #登録されたインシデント(トラブル報告)のデータを保存するデータベースファイル(SQLite)
├── init.sql               #データベース(MySQL)の初期設定およびテーブル作成を行うためのSQLスクリプトファイル
└── requirements.txt       # システム実行に必要な外部ライブラリのリスト


## 技術スタック
* **フレームワーク**: PythonのWebフレームワーク(Flask)をベースに構築
* **データベース**: SQLite / MySQL (※初期化用 `init.sql` あり)

## Webアプリの起動方法

### 1. 仮想環境の作成と有効化
# 仮想環境を作成
python -m venv venv
(環境によっては python3 app.py)


# 仮想環境を有効化 

(Windows / Git Bash の場合)
source venv/Scripts/activate

(Ubuntu の場合)
source venv/bin/activate

※ 仮想環境に正しく入ると、画面左端に (venv) と表示されます。


## 2.必要なライブラリのインストール
pip install flask

## 3.アプリの起動
python3 app.py
(環境によっては python app.pyで起動)
