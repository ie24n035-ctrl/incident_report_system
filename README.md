# クライアント 05 — インシデント報告

中規模工場・安全管理担当

### 状況
インシデントを紙の用紙に記録してバインダーに保管している。

### 問題
誰も分析しない。同じ場所で同じ事故が繰り返されている。

### 依頼
「インシデントをデジタルで報告して、同じ問題が繰り返されていたら教えてくれるシステムがほしい。」

### ディレクトリ構成と役割

```text
├── static/
│   └── style.css          # 画面のデザイン・レイアウトを整えるCSSファイル
├── templates/
│   ├── add.html           # インシデントを新規登録・報告する画面. [インシデント]=重大な事件や事故（アクシデント）には至らなかったものの、業務の正常な進行を妨げたり、将来的に大きな問題に発展したりする可能性のある「予期せぬ出来事」や「トラブル」
│   ├── edit.html          # 過去のインシデント内容を編集・修正する画面
│   └── index.html         # インシデント一覧および重複アラートを表示するメイン画面
├── README.md              # 本書（プロジェクトの説明書）
├── app.py                 # データの処理・画面遷移・重複分析を行うアプリ本体（Python）
└── requirements.txt       # システム実行に必要な外部ライブラリのリスト


```
PythonのWebフレームワーク(Flask)をベースに構築


webアプリ開き方
①仮想環境を作る
python3 -m venv venv
もし、エラーが出たら、
sudo apt install python3-venv
を実行してから、もう一度
python3 -m venv venv
を実行。


②仮想環境を有効化
source venv/bin/activate
成功すると、先頭に(venv)


③Flaskを入れる
pip install flask


④アプリを起動する
python3 app.py
